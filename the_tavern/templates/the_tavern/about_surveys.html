{% trans 'About Surveys' %}

{% trans 'What is a Survey?' %}

{% blocktrans %} Surveys let you create questionnaires to gather feedback from the community, register players for tournaments, or coordinate scheduling availability. They can be public for everyone or private for specific players, Discord Guild members, or series participants. {% endblocktrans %}

{% blocktrans %} Surveys support a variety of question types, optional quiz scoring, waitlist management, and integration with the Series system for tournament registration and availability-based grouping. {% endblocktrans %}

{% trans 'Question Types' %}

{% blocktrans %} Surveys support 12 different question types to capture a wide range of information: {% endblocktrans %}

{% trans 'Multiple Choice' %}
{% trans 'Select a single option from a list of choices.' %}
{% trans 'Multiple Selection' %}
{% trans 'Select one or more options from a list of choices.' %}
{% trans 'Open Ended' %}
{% trans 'Provide a free-text response.' %}
{% trans 'Ranking' %}
{% trans 'Drag and drop choices to order them by preference.' %}
{% trans 'Scale' %}
{% trans 'Rate on a numeric scale (e.g. 1-5 or 1-10) with labels.' %}
{% trans 'Yes / No' %}
{% trans 'A simple yes or no question.' %}
{% trans 'Date' %}
{% trans 'Pick a date from a calendar.' %}
{% trans 'Time' %}
{% trans 'Pick a time.' %}
{% trans 'Date & Time' %}
{% trans 'Pick both a date and a time.' %}
{% trans 'Time Availability' %}
{% trans 'Select which hours of the day you are available, organized by day of the week. Used for scheduling within a Series.' %}
{% trans 'Day Availability' %}
{% trans 'Select which days of the week you are available. Used for scheduling within a Series.' %}

{% blocktrans %} Root Component choices: Multiple Choice, Multiple Selection, and Ranking questions can use Root Database content (such as Factions, Maps, or Decks) as answer options. You can include all official components of a type or select individual items. {% endblocktrans %}

{% trans 'Survey Settings' %}

{% blocktrans %} Surveys offer several configurable settings to control how they behave: {% endblocktrans %}

{% trans 'Visibility' %}
{% blocktrans %} Public surveys are accessible to all logged-in users. Private surveys are restricted to invited players, Discord guild members, or participants of a linked Series. {% endblocktrans %}
{% trans 'Timing' %}
{% trans 'Set optional start and end dates to control when the survey is open for responses.' %}
{% trans 'Responses' %}
{% trans 'Choose whether users can submit multiple responses or edit their existing response while the survey is open.' %}
{% trans 'Results' %}
{% trans 'Control whether respondents can see results immediately after submitting, or only after the survey closes.' %}
{% trans 'Quiz Mode' %}
{% trans 'Enable quiz mode to set correct answers for questions. You can show scores and correct answers to respondents after they submit.' %}
{% trans 'Waitlist' %}
{% trans 'Set a cap on accepted responses. Once the threshold is reached, additional responses are placed on a waitlist.' %}
{% trans 'Sections' %}
{% trans 'Organize questions into named sections to break up longer surveys into logical groups.' %}
{% trans 'Series Registration' %}

{% blocktrans %} Surveys can be linked to a Series or Series Stage to support tournament registration and player scheduling. {% endblocktrans %}

{% trans 'Registration Mode' %}
{% blocktrans %} When enabled, the survey displays the tournament's rules and requires respondents to agree before submitting. This can be combined with a waitlist to cap the number of accepted registrations. {% endblocktrans %}
{% trans 'Availability Sync' %}
{% blocktrans %} Time Availability and Day Availability responses can be sent to the linked Series roster. This data is used for scheduling and grouping players based on when they are available to play. {% endblocktrans %}
{% trans 'Private Access' %}
{% blocktrans %} A private survey linked to a Series automatically restricts access to players or participants of that Series. {% endblocktrans %}
